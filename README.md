# Buildroot COMP8051 2026

Minimal Buildroot-based Linux environment for the COMP8051 Operating Systems module (Q5 setup).

## Overview

- **Buildroot version:** 2025.11.1  
- **Target architecture:** x86 (QEMU, i686)  
- **Linux kernel version:** 6.12.47 (from Buildroot)  
- **Use case:**  
  - Build a minimal Linux system for QEMU.  
  - Use it later for kernel experiments in Q6/Q7 (adding a custom syscall, tracing, etc.).

This repository tracks the configured Buildroot tree. Build artifacts (`output/`, `dl/`) are excluded via `.gitignore`.

---

## 1. Prerequisites (host system)

On the host Ubuntu VM, install the required packages:

```bash
sudo apt update

sudo apt install qemu-system-i386

sudo apt install -y   build-essential   bc   bison   flex   libssl-dev   libelf-dev   dwarves   pahole   rsync   cpio   kmod   libncurses5-dev   libncursesw5-dev
```

---

## 2. Initial Buildroot configuration

From the root of this repository:

```bash
cd buildroot-2025.11.1

# Base configuration for x86 QEMU target
make qemu_x86_defconfig
```

This generates `.config` for Buildroot with a minimal x86 system suitable for QEMU.

---

## 3. Linux kernel configuration (network drivers)

Configure the Linux kernel that Buildroot will build:

```bash
make linux-menuconfig
```

In the kernel menu, enable the following Intel network drivers as **modules**:

1. `Device Drivers  --->`
2. `Network device support  --->`
3. `Ethernet driver support  --->`

Set these entries to `<M>`:

- `Intel(R) PRO/100+ support (E100)`
- `Intel(R) PRO/1000 Gigabit Ethernet support (E1000)`
- `Intel(R) PRO/1000 PCI-Express Gigabit Ethernet support (E1000E)`

Then:

- Save the kernel configuration (`.config`),
- Exit back to the shell.

This ensures that the QEMU `e1000` device will have a working driver inside the guest.

---

## 4. Full Buildroot build

From the same directory:

```bash
make
```

This will:

- Download the Linux kernel sources and other packages.
- Build a cross-toolchain for i686.
- Build BusyBox and the root filesystem.
- Produce the kernel image and root filesystem image.

When the build finishes, key artifacts are in:

```bash
output/images/
  ├── bzImage        # Linux kernel (i686)
  └── rootfs.ext2    # Root filesystem image
```

---

## 5. Running the system under QEMU

From `buildroot-2025.11.1`:

```bash
qemu-system-i386   -M pc   -kernel output/images/bzImage   -drive file=output/images/rootfs.ext2,index=0,media=disk,format=raw   -append "rootwait root=/dev/sda console=tty0 console=ttyS0"   -serial stdio   --netdev user,id=net0,hostfwd=tcp:127.0.0.1:3366-10.0.2.14:22   -device e1000,netdev=net0
```

Notes:

- `-kernel`: uses the kernel built by Buildroot.
- `-drive`: attaches `rootfs.ext2` as `/dev/sda` in the guest.
- `-append`: tells the kernel to wait for `/dev/sda` and use it as root FS; also enables console on both VGA and serial.
- `-serial stdio`: routes the guest serial console to the host terminal.
- `--netdev ... -device e1000,netdev=net0`: adds a user-mode network backend with an `e1000` device and forwards host TCP port `3366` to guest port `22` (for later SSH/SCP use).

On successful boot you should see a login prompt like:

```text
Welcome to Buildroot
buildroot login:
```

Login as:

```text
Username: root
Password: (leave empty)
```

---

## 6. Enabling networking inside the guest

Once logged in as `root`:

```sh
modprobe e1000
ifup eth0
ip addr
```

- `modprobe e1000` loads the Intel PRO/1000 driver module.
- `ifup eth0` brings up the `eth0` interface.
- `ip addr` lets you verify the interface and IP address.

Later (for Q6/Q7) you can use SSH/SCP from the host to the guest via the forwarded port:

```bash
scp -P 3366 somefile root@localhost:/tmp/
```

---

## 7. Rebuilding only the Linux kernel

After modifying the Linux kernel sources under:

```text
output/build/linux-6.12.47/
```

you can rebuild just the kernel (without rebuilding the entire Buildroot system) from the Buildroot root directory:

```bash
make linux-rebuild
```

The updated kernel will be written again to:

```text
output/images/bzImage
```

You can then rerun QEMU with the same command as in section 5.

---

## 8. Notes

- Build artifacts (`output/`, `dl/`) are intentionally **not** tracked in Git to keep the repository small; they are generated by `make`.
- This setup is the foundation for later COMP8051 questions (Q6, Q7), where you will:
  - Modify `task_struct` to add tracing fields.
  - Add a new `trace_syscall` system call to the Linux kernel.
  - Rebuild the kernel and test the syscall from user space inside the QEMU guest.
